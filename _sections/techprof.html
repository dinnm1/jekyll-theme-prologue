As we were creating the modals and data display for every sea lion dynamically using multiple different functions, some buttons had to be created and appended before other elements. However, the team wanted certain buttons to appear at the very bottom of each column. To accomplish this I would normal use this CSS:

.button.submit_update
{
    position: absolute;
    bottom: 0;
}

However, that would place the button at the bottom right of the screen, whereas I needed it to be place at the bottom right of the div. It was extremely difficult to find any information on doing this within a div. Eventually I found this post that explained that I needed to add a relative position for the div that the button belongs to, like this:

.column_one
{
    position: relative;
}

.button.submit_update
{
    position: absolute;
    bottom: 0;
}

Checkbox: In the final stages of the project, when we were working on cleaning up the website, we decided the way we were displaying clipped toes in the modal was not very elegant. So I had to spend a lot of time recreating the checkbox design all in the JavaScript but only with the label so that is not click-able. var leftkeys = ["left1", "left2", "left3", "left4", "left5"];
var rightkeys = ["right1", "right2", "right3", "right4", "right5"];

var leftTitle = document.createElement("p");
leftTitle.innerHTML = "Left Clipped:";
col3cell.appendChild(leftTitle);

for (var i = 0; i < leftkeys.length; i++)
{        
    if(sealion[leftkeys[i]] == true)
    {            
        appendLabel = createLabel(i+1, true);
    }
    else if (sealion[leftkeys[i]] == false)
    {
        appendLabel = createLabel(i+1, false);
    }
    col3cell.appendChild(appendLabel);
}

function createLabel(num, clipped)
{   
    let label = document.createElement('label');
    if(clipped == true)
    {
        label.className = "update-clipped toe";
    }
    else
    {
        label.className = "toe-label toe";
    }
    label.innerHTML = num;
    
    return label;
}

Firestore -


Firestore is a flexible and scalable NoSQL cloud database that we used to store and sync data for our sea lions. This was the first time I have had any experience with a NOSQL database, so it was quite interesting to see the different ways of storing and retrieving data. The database model was completely different to what we’re typically used to, but it seemed to work for the case of the sea lions as there was no relationships needed, so we went with it.
